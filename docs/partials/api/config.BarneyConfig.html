BarneyConfig
service in module config

Description

To use Config service, you have to add BarneyConfig dependency to your component (directive, controller...). In this example, I have added dependency of BarneyConfig to a controller: 

angular.module('mock').controller('HomePageController', [
    'BarneyConfig', '$scope',
    function(Config, $scope){
        // we can use "Config" object here
    }
]);
Note that I included BarneyConifg as dependency but I have renamed it as Config to use it more easily in controller code.

List Methods:

  • get
  • init
  • list

Methods

  • get(key)

    This method is used to get the value of a specific configuration key

    Parameters
    ParamTypeDetails
    keystring

    Specific key

    Example

    Config get

    Here is an example of get method. 

    // Config.init({ 'ENABLE_LOGIN': 1 })

Config.get('ENABLE_LOGIN');

// 1

  • init(options)

    This method requires an object that contains all your keys and their values to initialize the Config service.

    Parameters
    ParamTypeDetails
    optionsObject

    config object

    Example

    Config Init

    Here is an example of init method. 

     Config.init(
     config: {
         'ENABLE_LOGIN': 1,
         'NEWTON_SECRET_KEY': '<sec_ret>',
         'KEY_TRUE': true,
         'KEY_VOID_STRING': '',
         'KEY_ZERO': 0,
         'KEY_ZERO_STRING': '0',
         'KEY_NULL': null,
         'KEY_NULL_STRING': 'null',
         'KEY_FALSE': false,
         'KEY_FALSE_STRING': 'false'
      }
     });

  • list()

    This method return the Configuration list.

    Example

    Conifg list

    Here is an example of how to use list method. 

    // Config.init({ 'ENABLE_LOGIN': 1 })

Config.get('ENABLE_LOGIN');

// { 'ENABLE_LOGIN': 1 }
