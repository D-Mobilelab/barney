storage

Description

Storage is the module for all the type of storage.

You can use it to:

  • set/get a single storage value;
  • set/get multiple storage values;
  • delete a storage value
  • check if localStorage, Cookie or JsObject storages are supported.

Import & Usage

Vanilla JS

Import this single file to use Storage module, in Vanilla JS: 

<script type="text/javascript" src="./bower_components/barney/base/storage.js">
Now you can use global object barney.Storage and associated methods, described below.
You can start by initializing it with init method.

Angular

Import this single file to use Storage module, in Angular framework: 

<script type="text/javascript" src="./bower_components/barney/angular/storage.js">
Now you can import barney module for your Angular app. 
angular.module('yourApp', [ 'barney' ]);
Storage module for Angular contains only BarneyStorage service.

Methods

  • delete(key, options)

    This method is used to delete a stored value in the default Storage.

    Parameters
    ParamTypeDetails
    keystring

    Key to identify value

    optionsObject

    (see param below)

    PropertyTypeDetails
    typestring

    If options.type is defined, it deletes the key from defined storage, else it deletes it from the default storage (defined in init method before)

    (default: null)

    Example

    Storage delete

    Here is an example of the delete method.

    In this example I delete from cookie the key called 'key'; 

    barney.Storage.delete('key', {type: Cookie});

  • get(key, options)

    This method is used to get a value from the storage.

    Parameters
    ParamTypeDetails
    keystring

    Key to identify value

    optionsObject

    (see param below)

    PropertyTypeDetails
    typestring

    If options.type is defined, it gets value in the defined storage, else it gets in the default storage (defined in init method before)

    (default: null)

    Example

    Storage get

    In this example I get the key 'key' from cookie storage 

    barney.Storage.get('key', {type: 'cookie'});

  • getMultiple(keys, options)

    This method is used to get multiple values in the Storage.

    Parameters
    ParamTypeDetails
    keysarray

    Array with keys to identify values

    optionsObject

    (see param below)

    PropertyTypeDetails
    typestring

    If options.type is defined, it gets values from the defined storage, else it gets from the default storage (defined in init method before)

    (default: null)

    Example

    Storage getMultiple
      barney.Storage.getMultiple(['key1', 'key2', ...], {type: 'cookie'});

  • init(params)

    Init storage module.

    Parameters
    ParamTypeDetails
    paramsObject

    (see attributes below)

    PropertyTypeDetails
    typestring

    Define the storage type used to save and get data. It can be:

    • localStorage,
    • cookie,
    • jsObject

    (default: null)

    loggerObject

    Object used to log (i.e: window.console, BarneyLogger, ...)

    (default: null)

    Example

    In this example I set cookie as default storage and BarneyLogger as default logger

     Storage.init({
   type: 'cookie',
   logger: BarneyLogger
});

  • isLocalStorageSupported()

    This method is used to know if the browser support the local storage

    Example

    if(barney.Storage.isLocalStorageSupported()) {
     // store a localstorage key
 } else {
     alert('local storage is not supported');
 }

  • set(key, value, options)

    This method is used to set a value in the storage.

    Parameters
    ParamTypeDetails
    keystring

    Key to identify value

    value*

    Value to store

    optionsObject

    (see param below)

    PropertyTypeDetails
    typestring

    If options.type is defined, it sets value in the defined storage, else it stores in the default storage (defined in init method before)

    (default: null)

    Example

    In this example I set the key 'key' with value 'value', in cookie storage 

    Storage.set('key', 'value', {type: 'cookie'});

    (see param below)

    PropertyTypeDetails
    typestring

    If options.type is defined, it sets value in the defined storage, else it stores in the default storage (defined in init method before)

    (default: null)
